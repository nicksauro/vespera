## 11. CRITICAL caveat — synthetic walk vs real WDO tape

**This section is mandatory per task brief §3 and Anti-Article-IV self-audit. Surfacing explicitly.**

### 11.1 What N6 actually executed

The events DataFrame consumed by `make_backtest_fn` carries **only metadata**: `(t_start, t_end, session, trial_id, entry_window)` per `build_events_dataframe` AC1 contract (T002.0f spec). It does **NOT** carry real WDO trade-tape data (parquet bars/ticks). This is by design per Mira spec §0 — Phase F downstream wires real tape replay.

For each event row, the closure body in `cpcv_harness.py:_walk_session_path` / `_walk_to_exit` evaluates the real strategy logic (T1..T5 entry rule per `compute_signal`, triple-barrier exits with SL>PT>vertical precedence per AFML 2018 §3.4, P60/P20/P80 magnitude/ATR filter resolution, cost atlas wiring) over a **deterministic synthetic per-event price walk**:

- Seed = SHA-256 hash of `(session, entry_window, trial_id, p126.as_of_date, split.path_id)` mixed via `numpy.random.default_rng(seed)`
- Walk parameters (literal constants in `cpcv_harness.py`):
  - `_SYNTH_OPEN_PRICE = 5000.0` (WDO mid baseline R$5000/USD nominal)
  - `_SYNTH_DAILY_VOL_POINTS = 25.0` (~25 points daily proxy)
  - `_SYNTH_15MIN_BARS_FROM_OPEN = 32` (09:30..17:30 = 32 × 15-min bars)
  - `_SYNTH_TICKS_BETWEEN_BARS = 8` (ticks per bar in barrier walk)

By construction (Dex commit body + module docstring): the synthetic walk is **strategy-logic-neutral** — zero edge expected, non-zero variance expected. K1/K2/K3 verdict semantics reflect the **strategy execution path** (entry rule + exit precedence + cost) over a noise process, NOT the real WDO market behavior.
