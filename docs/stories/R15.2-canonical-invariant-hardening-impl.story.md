## Problem framing

ADR-5 (`docs/architecture/ADR-5-canonical-invariant-hardening.md`) ratified the architectural extension of MWF-20260422-1 cosign discipline from the API write boundary (`materialize_parquet.py:_append_manifest`) to the **VCS write boundary** (`git commit`, `gh pr merge`). R15 established the *tracking baseline* (commit `ee786f80`); R15.2 *implements the enforcement*.

**Structural gap closed by this story:** `git add data/manifest.csv && git commit` does NOT transit `materialize_parquet.py`, so the existing API-side `MC-` flag enforcement has zero purchase on direct VCS mutations. Without R15.2, a developer (or a tool acting on their behalf — IDE re-save, CRLF normalization, OneDrive sync, antivirus quarantine-restore, etc.) can mutate canonical bytes and commit them with no audit trail. R15 closed the *tracking* gap (files now versioned); R15.2 closes the *write-discipline* gap (mutations now require an explicit cosign flag at the VCS boundary, defended by L1 pre-commit hook + L2 CI required check).

**Defense-in-depth model (per ADR-5 §3.2 + strategy §3 trade-off matrix):**
- **L1 pre-commit hook** — fast local feedback (~95% of mutations caught at developer's machine; skippable with `--no-verify`).
- **L2 CI required check** — server-side authoritative enforcement (catches drift even when L1 is bypassed; non-skippable when wired into branch protection).
- **L3 CODEOWNERS** — human review required for any change to enforcement infrastructure itself.

**Scope of this story:**
1. Provision `.github/canonical-invariant.sums` (sha-mirror canonical surface, derived per ADR-5 §5.2).
2. Provision `.githooks/lib/cosign-regex.sh` (single-source-of-truth regex extraction — ADR-5 §4.1 + Dex §12.2 row 4).
3. Provision `.githooks/pre-commit-canonical-invariant` (L1 enforcement hook with auto-update mechanism per C-2).
4. Provision `.github/workflows/canonical-invariant-protection.yml` (L2 CI required check per C-3 sequencing).
5. Provision smoke-test workflow validating known-bad commit fail-closed (per C-3).
6. Update `CODEOWNERS` (Dex + Riven for hook+workflow; Dara + Riven for `.gitattributes`; Gage for branch-protection scope).
7. Toggle workflow to required check on `main` branch protection (Gage R12 exclusive).
8. Quinn QA gate end-to-end.

**Out of scope (explicit, per ADR-5 §6 + Riven N-1/N-2):**
- Drafting `R1-1-Waiver-Spec` analogous to MWF-20260422-1 (Riven N-1; future, triggered when first non-step-7 `core/memory_budget.py` mutation is needed).

- [ ] **AC3** (encodes Riven C-1 — **R-B closure**) — `.githooks/pre-commit-canonical-invariant` implements **deterministic glob precedence first-match-wins** for ruling-document path resolution, **fail-closed on no-match**. Search order, in order, exactly:
  1. `docs/governance/${flag_lower}*.md` (per MC-29-1 R4 ruling precedent)
  2. `docs/qa/gates/${flag}-gate.md` (per MWF-22-1 gate precedent)
  3. `docs/architecture/memory-budget.md` with literal `grep -qF "$flag"` hit before accepting (Decision Matrix fallback)

  No-match across all three → `exit 1` (fail-closed; commit blocked). The reference impl pattern is the bash loop in ADR-5 §12.3 C-1 acknowledgment. R15.2 hook MUST honor this exact precedence; any deviation = Riven NO-GO at custodial review (T6).

- [ ] **AC4** (encodes Riven C-2 — **`.sums` auto-update hook-mandatory, NOT manual**) — The L1 pre-commit hook **auto-computes the new sha from the staged blob** (`git show ":$canonical" | sha256sum`) and **stages `.sums` atomically** alongside the canonical mutation. Reference impl pattern: ADR-5 §12.3 C-2 acknowledgment block (`mktemp`+`awk`+`mv`+`git add`). The hook MUST use `git show :path | sha256sum` (NOT on-disk `sha256sum path`) to read the staged blob — this is the invariant Dex empirically verified in ADR-5 §12.2 row 2 / §12.4. **Manual `.sums` edit without matching canonical mutation = L2 CI fails** (CI re-derives shas from canonicals, compares with `.sums`, sha mismatch fails the check). **Manual fallback explicitly DISALLOWED unless Riven re-cosigns** per Riven §11.3 escape clause — this story does NOT invoke the fallback (Dex §12.3 C-2 confirmed auto-update is implementable on Windows Git Bash).

**Hook decision tree (canonical mutation detected):**

3. Compute new sha from staged blob (`git show ":$canonical" | sha256sum`); update `.sums` line atomically; `git add .github/canonical-invariant.sums`.
4. `exit 0` (ALLOW).

**CI decision tree (PR or push to main):**
1. Checkout repo with `actions/checkout@v4 fetch-depth: 0`.
2. Run `sha256sum -c .github/canonical-invariant.sums`.
3. PASS → green check.
4. FAIL → red check (PR blocked by branch protection if T6 toggled).
