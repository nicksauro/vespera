## Architectural Notes (Aria T0a brief reference)

### State machine

```
pending ─[start]→ in_progress ─[validate PASS + manifest commit]→ ok
                       │
                       ├─[validate FAIL]→ partial
                       ├─[DLL error retryable]→ failed (retry < max)
                       ├─[DLL error terminal]→ retention_exhausted
                       └─[hard_timeout_s exceeded]→ failed (retry < max if transient)
```

### Module boundaries

```
scripts/dll_backfill_lib.py          # Pure library (no main)
├── ChunkPlanner                      # Calendar-aware chunk generation
├── ChunkSpec                         # Dataclass (start, end, ticker, exchange, chunk_id)
├── ChunkState (Enum)                 # State machine values
├── ManifestStore                     # Atomic manifest read/write/history
├── IncrementalParquetSink            # Streaming write 50k batch + JSONL sidecar
├── ValidateGate                      # 10 checks C1-C10
├── RetryPolicy                       # Transient vs terminal classification
├── DllSession                        # Init / login / finalize lifecycle
└── _cb_refs                          # Module-level GC anchor (Q08-E)

scripts/dll_backfill_orchestrator.py  # Main entry (idempotent loop)
scripts/dll_backfill_smoke.py         # Phase 2A wrapper (single chunk)
scripts/recover_parquet_from_jsonl.py # Sidecar → parquet recovery
scripts/generate_backfill_report.py   # Reconciliation reporter
```

### Recovery flow

1. Crash detected (orchestrator restart).
2. ManifestStore.read() → identify chunks `in_progress` (orphan from crash).
3. For each orphan: check sidecar `{chunk_id}.jsonl` exists with non-zero lines → if YES, run `recover_parquet_from_jsonl.py` → mark `partial` (validate gate decides ok/partial); if NO, mark `pending` → re-fetch from DLL.
4. Resume normal loop.
