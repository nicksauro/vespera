## User story

Como Beckett gerando o `T002-cpcv-report.md` na Fase E, preciso de um módulo canônico `packages/vespera_metrics/` que calcule **todas** as métricas pré-registradas na spec T002 v0.2.0 §metrics_required (primary: `IC_spearman`, `DSR`, `PBO`; secondary: `sharpe_distribution_45paths`, `sortino`, `MAR`, `max_drawdown`, `hit_rate`, `profit_factor`, `ulcer_index`, `bootstrap_ci_ic_10k`), cada uma validada contra reference value conhecido, para que os kill criteria K1 (DSR<0), K2 (PBO>0.4), K3 (IC_holdout vs in_sample) sejam avaliados com matemática correta — falso-positivo mata a tese, falso-negativo autoriza capital real sobre edge inexistente.

### API & escopo

- [ ] **AC1** Módulo `packages/vespera_metrics/` com subpacotes:
  - `vespera_metrics.info_coef` — `ic_spearman(predictions, labels) -> float`, `bootstrap_ci(ic_values, n=10000, confidence=0.95, seed=42) -> tuple[float, float]`
  - `vespera_metrics.sharpe` — `sharpe_ratio(returns, freq='daily') -> float`, `sharpe_distribution(paths_returns: list[list[float]]) -> np.ndarray` (shape (45,))
  - `vespera_metrics.dsr` — `deflated_sharpe_ratio(sr_observed, sr_distribution, n_trials, skew=0, kurt=3) -> float` conforme **Bailey & Lopez de Prado 2014** — "The Deflated Sharpe Ratio: Correcting for Selection Bias, Backtest Overfitting, and Non-Normality"
  - `vespera_metrics.pbo` — `probability_backtest_overfitting(cv_results_matrix: np.ndarray) -> float` conforme **Bailey, Borwein, Lopez de Prado, Zhu 2014** — "The Probability of Backtest Overfitting"
  - `vespera_metrics.drawdown` — `max_drawdown(equity)`, `ulcer_index(equity)`, `mar_ratio(cagr, max_dd)`
  - `vespera_metrics.trade_stats` — `hit_rate(trades)`, `profit_factor(trades)`, `sortino_ratio(returns, freq)`
- [ ] **AC2** Todas funções são **puras** (zero I/O, zero logging, zero globals) — consumíveis de qualquer contexto, determinísticas.

### Reference values (correctness)

- [ ] **AC3** **DSR test:** reproduz valor do toy example de Bailey & Lopez de Prado 2014 §3 (ou replica exemplo da implementação pública em [hudson-and-thames/mlfinlab](https://github.com/hudson-and-thames/mlfinlab) — Mira confirma referência) dentro de tolerância `1e-6`.
- [ ] **AC4** **PBO test:** reproduz a frequência de overfitting de um toy 2x4 matrix onde in-sample rank de cada strategy é negativamente correlacionado com out-of-sample rank — resultado esperado PBO ≈ 1.0 (Mira fornece valor exato na spec sign-off).
- [ ] **AC5** **IC Spearman test:** para `predictions = [1, 2, 3, 4, 5]`, `labels = [1.1, 1.9, 3.2, 3.8, 5.5]` → `ic_spearman == 1.0` (exato).
- [ ] **AC6** **Bootstrap CI test:** para sample `[0.1]*1000`, 95% CI ≈ `(0.1, 0.1)` com tolerância `1e-10` (sample sem variância).
- [ ] **AC7** **Sharpe test:** `returns = [0.01]*252` → `sharpe_annual ≈ sqrt(252) × 0.01 / 0` = `inf`; para `returns = [0.01, -0.005]*126` → calcular e bater com numpy reference.
- [ ] **AC8** **Sortino test:** returns só com downside não-zero usa std de downside; validar contra exemplo documentado.
- [ ] **AC9** **MAR test:** `MAR = CAGR / max_dd` com sinais corretos (CAGR>0, max_dd<0 ⇒ MAR<0 explícito, não NaN).
- [ ] **AC10** **Ulcer test:** validar contra Martin ulcer index reference (<https://www.tangotools.com/ui/ui.htm> formula).

### Determinismo

- [ ] **AC11** Bootstrap usa `numpy.random.Generator(PCG64(seed))` — seed passado explicitamente; 2 runs com mesmo seed ⇒ mesmo CI byte-a-byte.

### Integração CPCV

- [ ] **AC12** Função de alto nível `compute_full_report(cpcv_results: list[BacktestResult], n_trials: int, config: ReportConfig) -> FullReport` consome output de T002.0c `CPCVEngine.run()` e emite dict com todas métricas primary+secondary da spec.
- [ ] **AC12.1** **n_trials source-of-truth:** `compute_full_report` recebe `n_trials` **derivado de `docs/ml/research-log.md`** (cumulativo project-wide conforme AC0.1), **NÃO** apenas `spec.n_trials.total=5`. O `FullReport` output inclui campo `n_trials_source: "docs/ml/research-log.md@<git-sha>"` referenciando o log cumulativo. O relatório markdown de `to_markdown()` cita explicitamente o valor de `n_trials` usado e a fonte. Teste: `assert full_report.n_trials_source.startswith("docs/ml/research-log.md")`.
- [ ] **AC13** `FullReport.to_markdown()` emite relatório em formato esperado por `docs/research/results/T002-cpcv-report.md` (Beckett valida schema).

### Kill criteria wiring

- [ ] **AC14** Função `evaluate_kill_criteria(report: FullReport, thesis_kill_criteria: dict) -> KillDecision` que aplica K1, K2, K3 (K4 é paper-mode, fora de escopo aqui) conforme [thesis L126-146] e retorna `GO` ou `NO_GO` com reason explícita.
