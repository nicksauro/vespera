## Dex Notes (T1-T4 closeout, 2026-04-26 BRT)

### Empirical run finding вЂ” wall-time perf concern for Beckett T6 (AC8 < 30s budget)

Pre-handoff empirical validation of the streaming refactor was run with:

```
python scripts/run_warmup_state.py --as-of-dates 2025-05-31 --output-dir state/T002/
```

**Memory:** PASS вЂ” RSS held flat at ~120 MB throughout the entire orchestration window (poller telemetry: peak_wset 387 MiB, RSS sample stable 119-122 MB across all 15 polls). This is conclusive evidence that the streaming refactor works as designed and respects ADR-1 v3 CAP_v3 8.4 GiB (~1.4% of cap, ~7.9 GiB headroom for downstream CPCV harness).

**Wall time:** **6m05s observed** (well over Beckett's AC8 < 30s budget). Root cause analysis:

The per-day adapter call pattern mandated by AC2 (`source.load_trades(day_start, day_end, ticker)` per day, ascending) issues ~110 per-day parquet adapter calls for the 2025-05-31 lookback window. Each call:
1. Re-loads + parses `data/manifest.csv` + `data/manifest_PREVIEW.csv`
2. Filters to relevant monthly parquet file(s)
3. Verifies sha256 (cached after first open per file via `_INTEGRITY_CACHE`)
4. Opens `pyarrow.parquet.ParquetFile`, reads metadata, scans all row-groups via stats
5. Filters matching trades for the single-day window

The dominant cost is steps 4-5 multiplied by 110 days, NOT memory. The legacy single-call pattern paid these costs once; the streaming pattern pays them per-day by spec.

**This is NOT a Dex T1-T4 deliverable issue** вЂ” AC1-AC7 are about memory streaming, not wall time. AC8 belongs to Beckett T6 and likely needs an adapter-level optimization (Guard #4 places the adapter outside Dex's authority for this story).

**Suggested mitigation paths for Beckett T6 / Aria mini-council:**
1. **Adapter-level batching** вЂ” `feed_parquet.load_trades` could expose a per-day or per-month iterator that opens the parquet file ONCE and yields per-day groups. Maintains AC2 spirit (orchestrator iterates per-day) while amortizing parquet open + metadata cost.
2. **Hybrid per-month outer + per-day inner** вЂ” Re-interpret AC2 to allow per-month adapter calls, with the orchestrator splitting the stream per-day before reduction. Memory still bounded (no per-day list retention) but parquet I/O amortized 22:1.
3. **Open-once + iter_batches** вЂ” Within `_iter_parquet_rows` (adapter), keep an LRU-cached `ParquetFile` handle per path so 110 per-day calls re-use one open file descriptor.

All three above are **OUT OF SCOPE for T002.0h** (Guard #4 + AC2 literal interpretation). Surfaced to Beckett T6 / Aria mini-council for AC8 satisfaction.
