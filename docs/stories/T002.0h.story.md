## User story

Como **Beckett executando T11.bis smoke retry**, preciso que orchestrator T002.0g materialize warmup state JSONs SEM exceder ADR-1 v3 CAP_v3 8.4 GiB através de streaming per-day aggregation que descarte trade objects após DailyOHLC computation, preservando Mira anti-leak invariants (shifted-by-1, ascending iteration order). Sem este patch, `_aggregate_daily_with_close_at` (orchestrator.py:640-642) feeds `:334-336` (`buckets.setdefault(day, []).append(tr)`) retém TODOS os Trade objects do lookback 146bd em RAM (~3.7 GB residente), fazendo SIGTERM em 120s e bloqueando Phase E. Story-exit gate é Beckett T11.bis re-run PASS com peak RSS < 6 GiB.

## Anti-Article-IV Guards (5 fail-closed prescriptions)

Story body explicitly mandates Dex behavior; precedent T002.0g §3.6 5-guard pattern preserved:

1. **Dex MUST NOT extend timeout** para mascarar memory bug — fix root cause (single-pass aggregation). Beckett T11 timeout 120s permanece; AC8 exige < 30s warmup + < 5min CPCV smoke. Improvise extending timeout = Article IV violation.
2. **Dex MUST NOT raise CAP_v3 8.4 GiB** — ADR-1 v3 invariante imutável. Memory budget é design constraint, não tunável. Improvise raising cap = Article IV + ADR-1 violation.
3. **Dex MUST NOT subsample dataset** para passar AC6 — fixture é real-scale por design (146bd × ≥50k trades/day mínimo). Improvise subsampling para passar test = Article IV + Riven custodial violation (test masking).
4. **Dex MUST NOT mudar builder API** — Mira authority; mantém pure-compute via deque snapshots. `ATR20dBuilder` + `Percentiles126dBuilder` interfaces immutable. Improvise builder mutation = Mira anti-leak risk + R11 spec/CODE divergence.
5. **Dex MUST NOT skip ascending iteration** — Mira anti-leak conditional. Iteration ORDER strictly ascending (D-1 antes de D). Improvise reorder/parallel/forward-jump = anti-leak invariant violation (shifted-by-1 broken).
