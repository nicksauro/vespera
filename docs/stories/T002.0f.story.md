## User story

Como Beckett (@backtester, R6 CPCV authority) executando `*run-cpcv --dry-run --spec docs/ml/specs/T002-end-of-day-inventory-unwind-wdo-v0.2.0.yaml`, preciso de um **CPCV harness** que conecte os componentes já entregues por T002.0a-e + T002.1 — `{warmup gate → events_dataframe → backtest_fn (closure costs+calendar+P126 bands) → CPCVEngine 45-path × 5-trial fan-out → compute_full_report → KillDecision}` — para que eu produza um veredito **falsificável** sobre a tese T002, com `(T=5, N=45)` PBO matrix populada e K1/K2/K3 avaliados sobre dados reais (não stub), respeitando o protocolo de 6 GiB soft halt + telemetria psutil cadência ~30s. Sem este harness, todos os módulos T002.0a-e existem isolados e a Fase E não é executável.

## Acceptance criteria

### Harness module — events + backtest_fn factory (Beckett §1)

- [x] **AC1** `packages/t002_eod_unwind/cpcv_harness.py` exporta `build_events_dataframe(in_sample_start: date, in_sample_end: date, trials: tuple[str, ...]) -> pd.DataFrame` com:
  - Columns canônicas: `t_start, t_end, session, trial_id, entry_window`
  - **Warmup-gated:** chama `packages.t002_eod_unwind.warmup.gate.assert_warmup_satisfied(in_sample_start)` (T002.1) antes de emitir qualquer event; HALT se P126/ATR20 não estiverem prontos.
  - **Nova-session-validity gated:** chama validação de calendário Nova (rollover/half-day/holiday) — sessões inválidas para WDO são puladas (não emitidas como event).
  - Index monotonic increasing por `t_start` (engine `_validate_events` exige).
  - Dtype: `t_start, t_end` = `pd.Timestamp` BRT-naive (R2); `session` = `date`; `trial_id` ∈ `trials`; `entry_window` = string canônica per spec §trading_rules.
- [x] **AC2** `make_backtest_fn(costs: BacktestCosts, calendar: RolloverCalendar, percentiles_state: PercentilesState) -> Callable[[pd.DataFrame, pd.DataFrame, CPCVSplit], BacktestResult]` retorna **closure** sobre:
  - `costs` — `BacktestCosts.from_engine_config(docs/backtest/engine-config.yaml v1.0.0)` (lê Nova cost-atlas v1.0.0 ou TO-VERIFY default).
  - `calendar` — Nova rollover calendar (intra-fold session validity).
  - `percentiles_state` — **per-train-fold P126 bands** rebuilt for each split (NOT global P126 — leak vector). Closure encapsula `state` para que `backtest_fn` seja pure-function-style sem side-effects globais.
  - Closure retorna `BacktestResult` per fold conforme Beckett §1.1 shape (TradeRecord usa canonical `Fill` from `exec_backtest.py`).

### 5-trial fan-out (spec v0.2.0 §trading_rules)

- [x] **AC3** `cpcv_harness.py` orquestra **5-trial × 45-path fan-out** (T1-T5 declarados na spec v0.2.0):
  - Para cada `trial_id ∈ ('T1', 'T2', 'T3', 'T4', 'T5')`, executa `CPCVEngine.run(events_filtered_by_trial, backtest_fn)` → `list[BacktestResult]` len=45.
  - Total output: `dict[trial_id, list[BacktestResult]]` com 5 keys × 45 results = 225 fold-results.
  - Trial isolation: cada trial usa subset filtrado de `events` por `trial_id` column (não cross-contamination).
- [x] **AC4** **Determinism (carry-forward AC12 of T002.0c):** 3 runs do harness com mesmo seed + mesma window in-sample produzem 225 `BacktestResult` byte-identical (`content_sha256()` matching across runs).

### compute_full_report integration (Mira AC12/AC12.1/AC13 of T002.0d)

- [x] **AC5** `packages/vespera_metrics/report.py::compute_full_report(cpcv_results: dict[str, list[BacktestResult]], n_trials_source: str, config: ReportConfig) -> FullReport` implementado (atualmente DEFERRED stub em `report.py:6-11`):
  - Consome dict 5-trial × 45-path de AC3.
  - Constrói `(T=5, N=45)` PBO matrix per BBLZ 2014 (linha = trial, coluna = path) usando per-fold OOS Sharpe.
  - Chama `pbo.probability_backtest_overfitting` sobre matrix → escalar PBO global.
  - Calcula DSR via `dsr.deflated_sharpe_ratio` com `n_trials = read_research_log_cumulative()` (NOT spec.n_trials.total isolado — AC12.1 of T002.0d).
  - Retorna `FullReport(metrics=MetricsResult, per_path_results=tuple, kill_decision=evaluate_kill_criteria(...))`.
  - `n_trials_source` field populated com `docs/ml/research-log.md@<git-sha>` (AC12.1 carry).
- [x] **AC6** `compute_full_report` validado contra **synthetic fixture** reusando Mira toy benchmarks de [T002-vespera-metrics-spec.md](../ml/specs/T002-vespera-metrics-spec.md) §6 (T11 PBO 4×4 + T9/T10 DSR toys). Synthetic input: 5 trials × 45 fake `BacktestResult` com Sharpe distributions controladas → output `FullReport.metrics.pbo` e `.dsr` batem com Mira reference values dentro de `1e-6`.

### Dry-run CLI (Beckett invocation contract)

- [x] **AC7** `scripts/run_cpcv_dry_run.py` CLI com:
  - `--spec PATH` — required, path para spec yaml v0.2.0.
  - `--dry-run` — flag boolean (default true; `--no-dry-run` desabilita psutil polling para perf).
  - `--in-sample-start YYYY-MM-DD` `--in-sample-end YYYY-MM-DD` — window override (default lê da spec).
  - `--smoke` — flag boolean: 1-month subset (`in_sample_end - 30d`) antes de full run.
  - `--run-id STR` — override default uuid hex (Beckett T0: deterministic hash default em vez de uuid4).
  - `--seed INT` (default 42) e `--mem-soft-cap-gb FLOAT` (default 6.0) per Beckett T0 non-blocking adds.
  - Exit code 0 se PASS, 1 se HALT (mem soft-cap exceeded ou warmup failure ou hold-out violation ou smoke abort), 2 se determinism violation.
- [x] **AC8** **Memory protocol (feedback_cpcv_dry_run_memory_protocol.md):**
  - psutil polling cadência 30s (configurable `--mem-poll-interval-s`, default 30s).
  - **6 GiB soft halt threshold** (`process.memory_info().rss > 6 * 1024**3`) → set halt_event, emit warning row em telemetry.csv, persist current state, **HALT graceful** (não kill -9).
  - Telemetry CSV writer to `data/baseline-run/cpcv-dryrun-{run_id}/telemetry.csv` com columns SUPERSET (Pax gap #2): `(timestamp_brt, rss_mb, vms_mb, cpu_pct, peak_commit_bytes, peak_wset_bytes, sample_size, duration_ms, fold_index, trial_id, phase, note)` — satisfaz story E memory protocol.
- [x] **AC9** **Hold-out lock guard:** Antes de iteration, `scripts/run_cpcv_dry_run.py` invoca `_holdout_lock.assert_holdout_safe(in_sample_start, in_sample_end)` — fail-closed se window cruzar `[2025-07-01, 2026-04-21]` sem `VESPERA_UNLOCK_HOLDOUT=1` (R1+R15(d)).

### Smoke test

- [x] **AC10** `--smoke` run em **1-month subset** (configurable; default últimos 30 dias da `in_sample_end - 30d`) completa em <5min em hardware referencial (i5/16GB) sem exceder 6 GiB e produz `FullReport` válido (KillDecision verdict ∈ {GO, NO_GO}). _Smoke phase implementado; T11 Beckett re-run valida tempo real <5min com mock backtest._
- [x] **AC11** Smoke success é **pre-condition** documentada para 12-month full run; `--smoke` failure aborta full run automaticamente com exit code 1 + reason em telemetry CSV (`reason: smoke_aborted: <type>: <msg>`).

### Logging & traceability

- [x] **AC12** Run artifacts persisted em `data/baseline-run/cpcv-dryrun-{run_id}/`:
  - `telemetry.csv` (AC8)
  - `full_report.md` (`FullReport.to_markdown()` output)
  - `full_report.json` (FullReport serialized)
  - `determinism_stamp.json` (DeterminismStamp from BacktestRunner)
  - `events_metadata.json` (in_sample window + n_events + n_trials + warmup gate timestamp)
- [x] **AC13** Spec lock: `DeterminismStamp.spec_sha256` matches sha256 do `--spec` arquivo (Beckett §5.4); mismatch entre runs (mesmo `--run-id`) aborta com exit code 2.
