## Testing strategy

**Fixtures sintéticos:**
- 30 dias de trades com ret_dia conhecido → ATR_20d calculável à mão (ground truth)
- 126 dias de magnitudes sintéticas com percentis conhecidos (P20=0.2, P60=0.6, P80=0.8)
- Dia de vencimento (D-3..D-1) → garantir exclusão
- Dia de Copom → garantir exclusão do sample
- Feriado BRA → garantir exclusão

**Edge cases:**
- State file ausente → `WARM_UP_IN_PROGRESS` → builder roda → `READY_TO_TRADE`
- State file corrupto (JSON inválido) → `WARM_UP_FAILED` (não crasha)
- `GetHistoryTrades` retorna lista vazia → `WARM_UP_FAILED` + alerta
- Relógio do servidor antes de 09:00 BRT → warm-up pode rodar; após 09:30 ⇒ dia perdido
- Primeiro dia ever (< 20 dias úteis de histórico) → `WARM_UP_FAILED`

**Integração:**
- Rodar warm-up sobre 2025-Q1 contra dataset real → ATR_20d e percentis devem fazer sentido (comparar com cálculo manual em Excel)
