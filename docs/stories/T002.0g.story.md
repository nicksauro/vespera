## Risks

1. **R1 — Parquet source coverage gap:** `as_of=2024-06-30` exige lookback até ~2023-12-22. Verificar `data/in_sample/year=2024/` (e year=2023 boundary) cobre OU fail-closed com explicit error.
   *Mitigation:* Pre-flight coverage assertion em `orchestrate_warmup_state` antes de adapter open; if window not covered → `InsufficientCoverage` com window edges + missing dates listados (Article IV — no silent improvise). Dara T0 confirma coverage map.

2. **R2 — Path lift regression:** lift `data/warmup/` → `state/T002/` em 2 arquivos T002.0f pode quebrar testes existentes que assertam path string.
   *Mitigation:* T3 atomic com T1 no MESMO PR; espelhar override via `--output-dir` flag (R15 non-breaking — defaults overridable); rodar `pytest tests/cpcv_harness/ tests/scripts/test_run_cpcv_dry_run.py -q` antes do commit T3 e patch tests que hard-code path defaults.

3. **R3 — Determinism cross-run:** builder JSON serialization (dict keys order, float repr) deve ser sorted+canonical para garantir byte-equal.
   *Mitigation:* `json.dumps(..., sort_keys=True, separators=(',', ':'))`; floats via `repr()` round-trip-safe; AC7 test enforce; Mira approval do schema do `state_content_hash` em T0 handshake.

4. **R4 — Memory: 126bd × N as_of_dates pode estourar 6 GB se carregar tudo em memória:** streaming/chunked loader obrigatório.
   *Mitigation:* AC10 streaming per as_of_date (não merge global em RAM); MemoryPoller cadência 30s + soft halt graceful; if smoke run em hardware ref aborta por mem → optimize com dask/pyarrow batched antes de full Phase E.

5. **R5 — Schema drift:** T002.1 dataclasses evoluem → builder output desincroniza com consumers.
   *Mitigation:* AC8 schema validation com fixtures congelados em `tests/warmup_orchestrator/fixtures/`; cross-ref T002.1 dataclasses imports em orchestrator (single source of truth); CI fixture re-bake gated por Mira sign-off.
