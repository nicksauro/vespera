### Key technical decisions

1. **Calendar param injection (Mira T0)** — `orchestrate_warmup_state(calendar=CalendarData)` is REQUIRED kwarg (no default). Negative test `test_calendar_param_required_no_internal_instance` proves the orchestrator never instantiates a calendar internally. The same `CalendarData` instance is passed by reference to `ATR20dBuilder` + `Percentiles126dBuilder` (`test_calendar_param_propagated_to_builders` asserts identity).
2. **COPY not symlink (Beckett T0)** — `_copy_to_latest` uses `shutil.copyfile` (Windows-safe). Failure surfaces as `OSError` and aborts the run (Guard #2 fail-closed).
3. **Silent-fallback removal (Guard #1)** — `run_cpcv_dry_run._load_warmup_state` now raises `FileNotFoundError` if the file is missing, `ValueError` if corrupt or schema-drifted. Two regression tests in T6 protect this contract. Main loop maps both to exit 1 with explicit diagnostic ("Run scripts/run_warmup_state.py --as-of-dates <D> to materialize").
4. **tradeType escalation method (Guard #4)** — Orchestrator detects via `hasattr(trade_sample, "tradeType")` runtime probe; if missing, returns `TradeFilterDecision(status="escalated_adapter_gap", note="[USER-ESCALATION-PENDING] ...")` and continues with unfiltered trades. WARN log emitted; manifest.json line records `tradetype_filter_status="escalated_adapter_gap"` for traceability. Riven T8 reviews whether this blocks DoD.
5. **Determinism via canonical JSON** — `_canonical_json_bytes` uses `json.dumps(sort_keys=True, separators=(",", ":"))`. State files are written as canonical bytes (not pretty-printed) so the file SHA matches `state_content_hash`. AC7 3-run test passes byte-identical.
6. **Anti-leak window safety net** — `compute_window` walks back day-by-day until `WARMUP_VALID_DAYS_REQUIRED=146` valid sample days are found; if `CALENDAR_BUFFER_RATIO=2.5×` is exhausted without satisfying the count, raises `InsufficientCoverage` (Guard #1). Builders enforce strict `< as_of_date` (Guard #3) — current day NEVER enters the window.
7. **`run_warmup_state.py` early hold-out check** — In addition to the orchestrator's per-as_of_date `assert_holdout_safe`, the CLI runs the same check at top-level for ALL as_of_dates BEFORE invoking `orchestrate_warmup_state`. Operator sees the violation BEFORE any IO/memory work. Test `test_holdout_lock_guard_fires` asserts orchestrator was NOT called when guard fires.
