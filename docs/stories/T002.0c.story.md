## User story

Como Beckett executando a Fase E, preciso de um engine **Purged Combinatorial Cross-Validation** (Lopez de Prado, *Advances in Financial Machine Learning* Ch.12) parametrizado pela spec T002 v0.2.0 (N=10 grupos, k=2 test groups per split, 45 paths = C(10,2), `embargo_sessions=1`), seed-determinístico e parallel-safe, para que eu rode `*run-cpcv --spec T002-v0.2.0.yaml` e obtenha 45 backtest paths sem information leakage entre train/test e sem overlap temporal induzindo PBO falso.

---

## Acceptance criteria

### Engine core

- [ ] **AC1** Módulo `packages/vespera_cpcv/__init__.py` exporta:
  - `CPCVConfig(n_groups: int, k: int, embargo_sessions: int, seed: int)`
  - `CPCVEngine(config: CPCVConfig)`
  - `CPCVEngine.generate_splits(events: pd.DataFrame) -> Iterator[CPCVSplit]`
  - `CPCVSplit(path_id: int, group_ids_test: tuple[int, ...], train_events: pd.DataFrame, test_events: pd.DataFrame)`
- [ ] **AC2** Para `n_groups=10, k=2`, `generate_splits` produz exatamente `C(10, 2) = 45` `CPCVSplit`s com `path_id ∈ [0, 44]`
- [ ] **AC3** Grupos são particionamento contíguo temporal de `events` (não aleatório) — critério padrão AFML §12.3
- [ ] **AC4** **Purging:** para cada split, eventos de treino cujo `t_end` (label deadline) cai dentro do test window são removidos (AFML §7.4.1 + §12.4). Implementar via `_purge(train, test)`.
- [ ] **AC5** **Embargo:** após cada test group, `embargo_sessions=1` sessão adicional de treino é removida para impedir leakage por autocorrelação (AFML §7.4.2). Implementar via `_embargo(train, test_end, embargo_sessions)`.

### Determinismo e paralelismo

- [ ] **AC6** Seed-determinístico: mesmo `seed` + mesma `events` ⇒ mesma sequência de 45 splits, byte-a-byte (hash de cada split idêntico entre execuções)
- [ ] **AC7** Parallel-safe: `CPCVEngine` é stateless pós-construção; splits podem ser consumidos em paralelo via `joblib.Parallel` ou `multiprocessing.Pool` sem race condition (AC9 valida)

### Benchmark reprodutível

- [ ] **AC8** Unit test `test_cpcv_against_afml_benchmark.py` reproduz o benchmark numérico de AFML Ch.12 §12.4 Figure 12.2/12.3: dado um toy dataset de N=6/k=2 (= 15 paths), valores esperados de eventos purgados + embargados batem com a referência dentro de tolerância 0 (comparação exata).
- [ ] **AC9** Paralelismo test: `joblib.Parallel(n_jobs=4)(delayed(hash_split)(s) for s in engine.generate_splits(...))` produz lista de 45 hashes idêntica a versão serial.

### Integração spec T002

- [ ] **AC10** `CPCVConfig.from_spec_yaml(path)` constrói config lendo `cv_scheme` da spec T002 v0.2.0 L147-154 (n_groups=10, k=2, embargo_sessions=1, seed via `SPEC_SEED` env ou default 42). Validation: rejeita spec com `cv_scheme.type != "CPCV"`.
- [ ] **AC11** Interface para Beckett: `CPCVEngine.run(events, backtest_fn: Callable[[train, test], BacktestResult]) -> list[BacktestResult]` — shape consumível pelo `*run-cpcv` de Beckett (45 results, 1 por path).

### Safety

- [ ] **AC12** Se `events` contém qualquer timestamp ∈ `[2025-07-01, 2026-04-21]` e `VESPERA_UNLOCK_HOLDOUT != 1`, `generate_splits` lança `HoldoutLockError` antes de retornar qualquer split.
- [ ] **AC13** Zero lookahead interno: purging + embargo validados por assertions em runtime (train.index.max() < test.index.min() - embargo OR train.index.min() > test.index.max() + embargo).

  - `CPCVEngine(config)` com `np.random.Generator(PCG64(config.seed))` (determinismo AC6)
  - `_partition_groups(events)` — split contíguo em `n_groups` via `np.array_split(events.index, n_groups)`

  - `generate_splits(events)` — yield `CPCVSplit` com `path_id` = índice na ordem de `combinations`
  - `run(events, backtest_fn)` — consome `generate_splits`, invoca `backtest_fn(train, test)` por split, retorna `list[BacktestResult]` (len=45)

## Dev notes

- **Referência canônica:** Lopez de Prado, Marcos. *Advances in Financial Machine Learning*, Wiley 2018, Ch. 7 (purging/embargo) + Ch. 12 (Combinatorial CV).
- **Spec T002 v0.2.0 L147-154:** `cv_scheme.type=CPCV, n_groups=10, k=2, n_paths=45, embargo_sessions=1, purging=true`
- **R6:** determinismo (seed) é invariante — CPCV sem determinismo é PBO falso.
- **R1 + R15(d):** hold-out intocado (AC12).
- **Não usar scikit-learn `KFold`** — não implementa purging nem embargo.
- **Mira spec alignment:** Mira valida fórmula de purging + embargo contra seu entendimento do spec antes de Dex implementar.
- **Beckett interface:** shape de `BacktestResult` + `run()` negociado com Beckett no handshake (para `*run-cpcv` consumir direto).
