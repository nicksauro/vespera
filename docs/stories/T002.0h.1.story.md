## User story

Como **Beckett executando T11.bis Nx em Phase F com smoke + full phases potencialmente requiring distinct as_of dates**, preciso que o harness CPCV dry-run resolva warmup state **per-phase** usando paths **dated-by-as_of** (`atr_20d_{as_of}.json` + `percentiles_126d_{as_of}.json`) em vez de **default-path-singleton-with-last-write-wins** (`atr_20d.json` + `percentiles_126d.json`), de forma que smoke phase (`as_of = max(in_sample_start, in_sample_end - 30d)`, e.g. `2025-05-31`) e full phase (`as_of = in_sample_start`, e.g. `2024-08-22`) possam ambas validar warmup gate sem **mutual collision**. Sem este patch, Beckett N4 demonstrou empiricamente que AC8 amended invocation com `--smoke` aborta `exit=1` em ~2.892s por strict-equality default-path mismatch (`atr: stale as_of_date (file=2024-08-22, expected=2025-05-31)`). ESC-010 Track A E2 narrowing (drop `--smoke`) é workaround de curto prazo; **E1 é o structurally correct fix** preservando Dara data-integrity stance (audit chain per-phase) + Mira anti-leak invariants + Riven custodial barrier (Gate 1 of §9 HOLD #2 disarm chain).

### Aria T1 Design Memo — 2026-04-28 BRT

**Scope:** T1 deliverable per task spec — design canon for Dex T2 implementation. Inline (no separate `docs/architecture/` file — keeps design canon co-located with story for traceability and Article IV ledger discoverability).

#### 1. Per-phase `WarmUpGate` constructor signature

**Decision:** **NO change to `WarmUpGate.__init__` signature.** Constructor remains `__init__(self, atr_path: Path, percentiles_path: Path) -> None` (per `packages/t002_eod_unwind/warmup/gate.py:48-50`). The architectural shift is at **call-site only** — `scripts/run_cpcv_dry_run.py::main` no longer instantiates a single shared `WarmUpGate`; each `_run_phase` call constructs a phase-local `WarmUpGate` against dated paths.

**Rationale:** Modifying `WarmUpGate.__init__` to take `as_of_date` would couple the gate to date-resolution logic (anti-pattern; gate is a pure path-bound state validator). Strict-equality semantics at `gate.py:106-107` (`if file_date != expected_date`) remain unchanged — gate continues to verify file's `as_of_date` matches the `as_of_date` arg passed to `check()`. Per-phase usage is achieved by binding **different paths** to different instances, not by changing the gate's API. Minimal blast radius. Preserves all existing `WarmUpGate` unit tests.

**Call-site delta (high-level pseudo-code, NOT prescriptive — Dex retains implementation discretion):**

```python
# BEFORE (:848 — singleton):
warmup_gate = WarmUpGate(args.warmup_atr, args.warmup_percentiles)
# ... and `warmup_gate=warmup_gate` passed into both _run_phase calls

# AFTER — singleton REMOVED at :848.
# Each phase derives as_of_date and constructs its own gate from dated paths.

# Smoke phase (around :864-879):
smoke_atr_path = _resolve_warmup_path(args.warmup_atr, _dated_atr_path(smoke_start))
smoke_pct_path = _resolve_warmup_path(args.warmup_percentiles, _dated_percentiles_path(smoke_start))
smoke_gate = WarmUpGate(smoke_atr_path, smoke_pct_path)
# pass `warmup_gate=smoke_gate` + `warmup_percentiles_path=smoke_pct_path` into _run_phase

# Full phase analogous (around :910-924):
full_atr_path = _resolve_warmup_path(args.warmup_atr, _dated_atr_path(in_sample_start))
full_pct_path = _resolve_warmup_path(args.warmup_percentiles, _dated_percentiles_path(in_sample_start))
full_gate = WarmUpGate(full_atr_path, full_pct_path)
```

**`_resolve_warmup_path` resolver protocol** (preserves CLI affordance):

```python
def _resolve_warmup_path(cli_arg: Path, dated_default: Path) -> Path:
    """If cli_arg is the legacy default constant, use the dated default;
    else honour operator override. Preserves CLI affordance for ad-hoc
    invocations (e.g., manual smoke tests with hand-crafted state files).
    """
    if cli_arg in (_DEFAULT_ATR_PATH, _DEFAULT_PERCENTILES_PATH):
        return dated_default
    return cli_arg
```

(Dex may simplify if argparse `default` sentinel detection is clearer — implementation detail. **Invariant:** gate semantics MUST resolve to dated path on default invocation; operator-override path MUST be honored verbatim.)

**Alternative shape (Dex may prefer):** push the resolver into `_run_phase` and make its signature `_run_phase(..., as_of_date: date, ...)` — derive paths internally. This eliminates 4 lines of duplicate prep code in `main`. **Recommendation:** this alternative shape (cleaner contract; gate construction is a `_run_phase` internal concern, not a `main`-level orchestration concern). Either is design-canon-compliant.

#### 3. `cache_audit.jsonl` phase-tag schema delta

**Existing schema** (`scripts/run_warmup_state.py:617-624`, producer-side):

```python
entry = {
    "computed_at_brt": datetime.now().isoformat(timespec="seconds"),
    "as_of_date": as_of.isoformat(),
    "status": status,                  # ∈ {hit, miss, stale, write, force_rebuild}
    "expected_key": expected_key,      # triple-key dict
    "found_key": found_key,            # triple-key dict | None
    "note": note,
}
```

**Post-T002.0h.1 schema** (additive — consumer-side emission):

```python
entry = {
    "computed_at_brt": datetime.now().isoformat(timespec="seconds"),
    "as_of_date": as_of.isoformat(),
    "status": status,                  # consumer-side status (suggested literal: "consumer_check")
    "expected_key": expected_key,
    "found_key": found_key,
    "note": note,
    "phase": phase,                    # NEW — ∈ {"smoke", "full"} | absent for legacy/producer entries
}
```

**Schema migration semantics (design canon):**

- **Producer side** (`run_warmup_state.py::_append_cache_audit`): does NOT emit `phase` field (orchestrator runs are not phase-bound — they materialize state files independent of consumer phase). **Recommendation:** producer keeps current schema; phase tag is consumer-emitted only.
- **Consumer side** (NEW emission in `scripts/run_cpcv_dry_run.py::_run_phase`): each phase emits **one** `cache_audit.jsonl` entry with `phase: "smoke"` or `phase: "full"`. Status field SHOULD describe consumer-side gate verdict (suggested literal: `"consumer_check"` for successful per-phase gate consumption — Dex finalizes wording, but it MUST NOT collide with existing producer status enum `{hit, miss, stale, write, force_rebuild}`).
- **Append-only invariant:** preserved (Sable consumer reads JSONL forward; legacy entries lack `phase` interpreted as `null`).
- **Backward-compat:** any downstream `cache_audit.jsonl` reader MUST tolerate missing `phase` field on legacy entries (defensive `.get("phase")` access pattern). This is a Sable-side documentation responsibility; design canon flags it explicitly here.
- **Implementation note for Dex:** consumer-side audit emission may either (a) reuse `_append_cache_audit` from `run_warmup_state.py` by importing it (and threading a new optional `phase: str | None = None` kwarg into the producer helper), or (b) inline a small consumer-only helper in `run_cpcv_dry_run.py` that writes the consumer-side entry.
  - **Recommendation:** option (b) — inline helper. Consumer audit semantics differ from producer audit semantics (consumer = gate-decision audit; producer = cache-state audit); separating them clarifies intent and avoids mutating `_append_cache_audit` signature (which is producer-side canonical).

#### 4. Default-path behavior post-fix (CLI affordance retention)

**`_DEFAULT_ATR_PATH` / `_DEFAULT_PERCENTILES_PATH` (`scripts/run_cpcv_dry_run.py:120-121`):** **RETAINED unchanged** as legacy constants. Their role becomes purely:

1. **CLI defaults** for `--warmup-atr` / `--warmup-percentiles` argparse args (operator-friendly so manual invocations don't require typing the dated path).
2. **Symlink/copy targets** (operator/ops responsibility, **OUT OF SCOPE for this story**) — operators may symlink or copy the most-recent dated state file to the default path for convenience tooling that doesn't pass `--warmup-*` overrides. NOT enforced by this story; flagged as future operator UX improvement.

**Critical:** **gate semantics NEVER read default-path under default invocation.** The resolver protocol (§1) detects whether `args.warmup_atr` equals the default constant and substitutes the dated path. If operator passes `--warmup-atr custom_state.json`, the override is honored verbatim (operator authority for ad-hoc state files).
