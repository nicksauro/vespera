## 2. Gaps detectados no processo

### G001 ⚠️ [GAP] Decisão Framework 4Q da Kira não é formalmente rastreada

- **Descrição:** Kira aplicou o 4Q (Q1-Q4) e anotou "SIM" em todas. O processo não tem template formal — texto livre no thesis doc. Se amanhã surgir conflito ("mas Q2 era realmente SIM?"), não há assinatura ou timestamp específico do momento da decisão 4Q.
- **Evidência:** thesis T001 §1 tem tabela mas não tem campo `decided_by: Kira` + `decided_at: timestamp`.
- **Ação sugerida:** Adicionar seção padrão no thesis-tmpl.yaml com campos estruturados para cada Q (pergunta, resposta sim/não, justificativa curta, decided_by, decided_at).

### G002 💡 [COSMETIC] Spec YAML não tem hash real em dry-run

- **Descrição:** `mira_signature: "dry-run-smoke-test-no-real-hash"` — ok para dry-run, mas processo canônico deveria calcular hash SHA256 do conteúdo (ex via `sha256sum file.yaml`) e reinjetar.
- **Ação sugerida:** Comando Mira `*export-spec` deve, no fluxo real, calcular hash e escrever no campo automaticamente.

### G003 ⚠️ [GAP] Nova consulta em Fase A não produz artefato próprio

- **Descrição:** No thesis T001 §7 "Consulta Nova" aparece como checkboxes, mas sem link para atlas/documento onde Nova registrou a validação. Rastreabilidade frágil.
- **Ação sugerida:** Nova deve produzir `docs/research/audits/T001-nova-audit.md` (pequeno) e thesis linka. Aplicável também a Mira §6 e Nelo §8.

### G004 💡 [COSMETIC] Handoff A→B não tem assinatura digital de gate

- **Descrição:** Kira diz "PASS — liberado para Fase B" mas não há campo tipo `gate_A_signed_by: Kira, timestamp: ...` que Beckett/Aria possam verificar programaticamente.
- **Ação sugerida:** Adicionar rodapé de gate-signature em cada thesis doc (ou criar `.aiox/handoffs/` artifact conforme protocol de handoff existente).

### G005 ⚠️ [GAP] Spec YAML tem `contract_multiplier_brl_per_point: 10` hardcoded

- **Descrição:** Mesmo citando glossário como single source, o número está duplicado no YAML. Se glossário mudar, spec fica stale.
- **Ação sugerida:** Ou remover do YAML (Beckett busca direto no glossário), ou gerar YAML a partir do glossário (import-de-config), ou adicionar validação CI que compara.
