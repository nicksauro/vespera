## §4 Latency wiring verdict — **PASS_WITH_CONDITIONS**

Mira §5 consumes Beckett latency spec verbatim. Architectural review of wiring mechanics:

1. **`enabled_for_phase: ["F"]` gating.** Beckett §4 yaml line `enabled_for_phase: ["F"]` is the **synthetic-skip gate**. Architecturally, this maps cleanly to current factory dispatch: the closure body checks `latency_model.enabled_for_phase` and short-circuits to legacy zero-latency path under non-F regimes (synthetic walk, tests). Mira §5.2 confirms `synthetic walk N6 events were instantaneous; latency physically n/a` — the gate IS the synthetic-skip. **PASS** structurally.

2. **Slippage integration formula idempotency.** Beckett §3.1 + Mira §5.3 specify additive composition: `total_slippage_pts = roll_spread_half_points + slippage_extra_ticks × tick_size + slippage_latency_pts`. This preserves engine-config v1.0.0 §slippage_model **without double-counting** — roll is the immediate cushion (executed once per fill), latency is temporal mid drift during round-trip (a distinct, additive term). Mira §5.2 row 2 explicitly flags this no-double-counting assertion. **PASS** structurally.

3. **Determinism — seed_source cross-platform stability.** Beckett §2.4 specifies `seed_event = hash((session, order_id, trial_id, component_name))` via `blake2b_64`. Mira §5.2 row 3 confirms "guarantees bit-identical replay, zero cross-event correlation, zero global RNG state leak." **PASS.** Note that Beckett spec §2.4 lists `seed_source: ["session", "order_id", "trial_id"]` in the yaml block while the seed derivation in §2.4 prose ALSO mixes `component_name` — the `component_name` is implicit per-component partition, not a yaml-listed seed dimension. This is a documentary asymmetry, not a bug.

**Condition C-A2 (binds T0c Beckett + T1 Dex):** Slippage integration MUST be placed at fill-event materialization time, not at PnL aggregation time, otherwise the seed `(session, order_id, trial_id, component_name)` cannot be deterministically anchored to the specific fill event. T0c Beckett MUST verify the `BacktestCosts.from_engine_config` (or sibling `LatencyModel.from_engine_config`) helper exposes a per-fill API rather than a per-trade aggregate.

**Condition C-A3 (binds T1 Dex + T2 Quinn):** `latency_model.enabled_for_phase` gate MUST be tested via at least one unit test asserting `enabled_for_phase=["F"]` triggers the lognormal draw path AND `enabled_for_phase=[]` (or any list excluding "F") returns `slippage_latency_pts == 0` deterministically. This test belongs in the Phase F real-tape test file (per AC6 of T002.6 story; new file `tests/cpcv_harness/test_toy_benchmark_real_tape.py` or sibling).

**Aria verdict on §4:** latency wiring — **PASS** with bindings C-A2, C-A3.

## §5 RLP + microstructure wiring verdict — **PASS_WITH_CONDITIONS**

Mira §4 consumes Nova RLP/rollover spec verbatim. Architectural review:

1. **RLP detection_signal_historical "GAP" — degraded behavior clean.** Nova §1.3 + Mira §4.3 explicitly document the historic parquet gap: `tradeType` enum ABSENT, only `aggressor ∈ {BUY, SELL, NONE}`. Mira §4.1 yaml encodes `detection_signal.historic_field: null` and §4.3 mandates determinism stamp records `data_quality.cross_trade_flag_available: false` and `data_quality.rlp_flag_available_historic: false`. Architecturally — when the discriminator is null, RLP fills silently disappear into the BUY/SELL aggressor stream and are treated as Option-B uniform-slippage fills (no instant-fill bonus). **No crash path; degraded behavior explicit and stamped.** **PASS.**

2. **Rollover flag preservation through factory closure.** Mira §4.1 yaml encodes `rollover.flag_d_minus_3_to_d_minus_1: true` per-session; Nova §2.4 Option B preserves sample with `rollover_window: bool` flag. Architecturally, this flag is a **per-session feature** that propagates through `train_events`/`test_events` columns into the closure body — the factory captures it via the standard fold partition mechanism, NO changes required to the factory signature. Carry-forward bug guard (§4.1 yaml `carry_forward_bug_guard: true`) is enforced at the harness level via Nova §2.5 mandate (resolve explicit contract code per session; no naive WDOFUT carry-forward). **PASS.**

3. **Auction 3.2-α exit price rule (last non-auction trade < 17:55:00 BRT).** Nova §3.2 + Mira §4.1 yaml encode `auction.exit_price_rule: "last trade_type in {1,2,3,13} with timestamp < 17:55:00 BRT"`. Architectural review: this rule is consumed at the **vertical-barrier exit** path (the 17:55 BRT clock barrier from spec yaml v0.2.3). It does NOT alter triple-barrier precedence (`SL > PT > vertical` per AFML 2018 §3.4) — it only refines the **price** at which the vertical barrier executes. Mira §2.1 reaffirms precedence verbatim. **No precedence corruption. PASS.**

4. **CB flag + cross_trade flag propagation.** Nova §4 + §5 + Mira §4.1 yaml encode per-session `circuit_breaker_fired: bool` and per-event `cross_trade: bool`. These propagate through `train_events`/`test_events` schema into the closure as additional columns. They feed the determinism stamp (§3.5) and Riven's 3-bucket attribution downstream. Architecturally — additive columns, no schema-breaking change. **PASS.**

**Condition C-A4 (binds T1 Dex):** the per-session `rollover_window` and `circuit_breaker_fired` flags MUST be added to `events_dataframe` schema **before** the engine partition step (i.e., they live alongside `(t_start, t_end, session, trial_id, entry_window)` per `build_events_dataframe` AC1). If they are added downstream of partition, the train/test slicing may drop them silently and the closure cannot consume them. Suggested integration site: `build_events_dataframe` body or a thin wrapper that joins session-level features post-construction.

**Condition C-A5 (binds T1 Dex):** `cross_trade` flag is per-EVENT (Nova §5.2), but in historic regime is unsettable (Nova §1.3). Dex MUST emit the column with `False` default for historic regime AND record `data_quality.cross_trade_flag_available: false` in determinism stamp; this avoids future Phase H regression where the flag suddenly populates and breaks downstream consumers expecting None.

**Aria verdict on §5:** RLP + microstructure wiring — **PASS** with bindings C-A4, C-A5.
