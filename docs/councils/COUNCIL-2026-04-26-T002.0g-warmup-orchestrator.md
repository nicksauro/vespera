**Source loading:**
- **AC1** — `packages/t002_eod_unwind/warmup/orchestrator.py::orchestrate_warmup_state(as_of_dates: list[date], source: ParquetSource, output_dir: Path)` lê de `data/in_sample/` via `feed_parquet.py` (T002.0b adapter REUSE) — **NÃO TimescaleDB, NÃO live DLL**.

**Hold-out defense (Riven authority):**
- **AC3** — Orchestrator chama `_holdout_lock.assert_holdout_safe(window_start, window_end)` ANTES de adapter open. Defense-in-depth com adapter T002.0b (já chama). Reject `as_of_date >= 2025-07-01` com explicit error até Fase E.

- **AC4** — Orchestrator filtra `tradeType ∈ {2, 3}` (CompraAgressao + VendaAgressao). Cross trades (1), Leilão (4), RLP (13), CrossTrade (TradeType.Cross) excluídos. SE T002.0b adapter já filtra, AC vira validation assert; SE NÃO filtra, escalate (não improvisar — gap upstream).

**Multi as_of_date (C7+C8):**
- **AC5** — `--as-of-dates 2025-05-31,2024-06-30` aceita lista; cada date vira call independente. Output: `state/T002/atr_20d_2025-05-31.json` + `percentiles_126d_2025-05-31.json` + `atr_20d_2024-06-30.json` + `percentiles_126d_2024-06-30.json`.
