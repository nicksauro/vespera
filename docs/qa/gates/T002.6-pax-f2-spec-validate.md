### §2.8 Point 8 — Risks documented

**Question:** §15.5 Anti-Article-IV Guard #8 + §15.6 `InvalidVerdictReport` raise mitigates inconclusive-masking risk?

**Verdict:** **PASS (1/1).**

**Evidence:**
- **Risk identified (Round 1 root cause):** Numeric default `0.0` propagated verbatim end-to-end through `ReportConfig` → `MetricsResult.ic_spearman` → `evaluate_kill_criteria` K3 numeric eval → `K3_FAIL` emission. The `K_FAIL` verdict was **invalid by construction** because no IC measurement was ever performed — the failure was a data-quality artifact, not a strategy-edge signal. Sable F-05 anti-pattern catalog candidate.
- **Mitigation (§15.5 Anti-Article-IV Guard #8 NEW):**
  - Every numeric metric serialized in a verdict-issuing `KillDecision` MUST carry a per-metric `*_status` provenance flag from `{computed, deferred, not_computed, inconclusive_underN}`.
  - Verdict layer (`evaluate_kill_criteria`) MUST consume the `*_status` flag and emit `K_NOT_COMPUTED` (or `K_DEFERRED` for hold-out-locked cases) rather than `K_FAIL` when the flag is anything other than `computed`.
  - A verdict report that emits `K_FAIL` while `*_status != "computed"` is **invalid by construction** and MUST raise `InvalidVerdictReport` before persisting `full_report.json`.
- **Enforcement (§15.6):** Branch tree in `evaluate_kill_criteria` makes the inconclusive-masking failure mode structurally impossible:
  - `ic_status == "computed"` → numeric K3 eval (honest path).
  - `ic_status == "deferred"` → `ValueError` (forbidden in Phase F2 binding measurement).
  - `ic_status ∈ ("not_computed", "inconclusive_underN")` → `InvalidVerdictReport` (short-circuits before `K_FAIL` emission).
- **Test coverage (§15.8 test 4):** `test_ic_inconclusive_path` asserts `evaluate_kill_criteria` raises `InvalidVerdictReport` rather than silently emitting `K3_FAIL`. Binary-verifiable at F2-T2/T3 Quinn QA gate.
