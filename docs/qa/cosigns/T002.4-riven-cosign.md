### R3. Q6d Nyquist immunity

Capture mechanism is **kernel peak retrieval at child process exit**, not sampling. Verified in `scripts/materialize_parquet.py`:

- `main()` wraps the legacy `parse_args/run` dispatch in `try/finally:`; the `finally:` block unconditionally invokes `_emit_child_peak_telemetry()` on every exit path (success, `HoldoutLockError`, generic `Exception`).
- `_emit_child_peak_telemetry()` calls `_collect_child_peak_memory()` which reads `psutil.Process().memory_info().peak_pagefile / .peak_wset` — these are Windows kernel counters (`PROCESS_MEMORY_COUNTERS.PeakPagefileUsage / .PeakWorkingSetSize`) that the kernel updates on every allocation spike. No sampling interval is involved; the kernel "polls" itself at allocation granularity.
- Emission uses `print(..., flush=True)` to stdout (wrapper redirects child stdout → `{run_id}.log`), then a defensive inner `try/except` ensures emission failure does not mask the child's real return code (AC-K(iii) fallback semantics preserved).
- Parent-side polling in `core/run_with_ceiling.py` is **untouched** (`git diff 80805ac^ 80805ac -- core/run_with_ceiling.py` empty) — AC-D preserved. The Nyquist-vulnerable parent-poll artifact that produced retry #5's 800KB ghost peak is **bypassed** by the new mechanism, not modified.
