## ¬ß1 ‚Äî Signal-to-trade conversion logic

**Source of truth:** spec yaml v0.2.3 L80-83 + L194-212 (n_trials T1..T5).

### ¬ß1.1 ‚Äî Per-event invocation contract

For each row `(t_start, session, trial_id, entry_window)` in the `test_events` DataFrame produced by `build_5_trial_events_df` (cpcv_harness.py:267-264), `make_backtest_fn`'s closure body executes **exactly one trade decision**:

```
Step 1. Resolve TrialParams ‚Üê _resolve_trial_params(trial_id)
Step 2. Compute Features    ‚Üê compute_features(...)        # pure
Step 3. compute_signal(features, atr_ratio_bands, trial)   # pure, signal_rule.py:51
Step 4. If Signal.direction == FLAT  ‚Üí emit no trade; record n_flat_signals++
        Else                          ‚Üí emit trade per ¬ß2 triple-barrier
```

**The closure NEVER iterates beyond a single (t_start, trial_id) row.** Cross-event accounting (sample uniqueness, Sharpe per fold) is the engine's responsibility, not the closure's.

### ¬ß1.3 ‚Äî Direction (fade)

Per spec yaml L83 (`direction: "fade"`):

## ¬ß2 ‚Äî Triple-barrier exit semantics

**Source of truth:** spec yaml L70-74 (`label_alt`).

### ¬ß2.1 ‚Äî Three barriers (Lopez de Prado AFML 2018 ¬ß3.4)

Given entry fill `(entry_ts, entry_price)`:

| Barrier | Formula | Trigger condition |
|---------|---------|------------------|
| **Profit target (PT)** ‚Äî upper for LONG, lower for SHORT | `pt_offset = 1.5 * ATR_hora` (spec L72) | First trade tick where `(side == LONG and tick_price >= entry_price + pt_offset)` OR `(side == SHORT and tick_price <= entry_price - pt_offset)` |
| **Stop loss (SL)** ‚Äî lower for LONG, upper for SHORT | `sl_offset = 1.0 * ATR_hora` (spec L73) | First tick where `(side == LONG and tick_price <= entry_price - sl_offset)` OR `(side == SHORT and tick_price >= entry_price + sl_offset)` |
| **Time horizon (vertical)** | `t_vertical = 17:55:00 BRT same session` (spec L74) | Hard stop if no PT/SL hit before `t_vertical`; mid-price at `t_vertical` is the fill price |

### ¬ß2.2 ‚Äî `ATR_hora` definition

Per spec yaml L72-73: `ATR_hora` is **hourly ATR** (not 20d daily ATR). Mira ruling resolves the ambiguity:

```
ATR_hora(session, t) = mean(true_range_15min[s, t-4..t])  # rolling last 4 √ó 15-min bars
```

where `true_range_15min` is computed over 15-minute bars from session open 09:30 BRT. This gives ~4 bars/hour, matching standard market-microstructure ATR practice.

**Computability against historical-availability (Nova trades-only constraint):** PASS ‚Äî true range from 15-min OHLC is computable from trade tape alone (no book required).

**Defensive fallback:** if `ATR_hora` is undefined (first hour of session, < 4 bars) ‚áí **DO NOT TRADE**. The signal_rule's existing `regime_atr_undefined` flag covers this; closure checks `Features.flags` and emits FLAT.

### ¬ß2.3 ‚Äî Barrier-hit precedence rules

When two barriers fire on the same tick (rare but possible on gaps):

```
Precedence: SL > PT > vertical
```

**Rationale (Mira primary literature anchor, Lopez de Prado AFML 2018 ¬ß3.4):** stop-loss first preserves risk-management invariant; PT second; vertical only fires when neither other barrier triggered. This is the canonical AFML ¬ß3.4 ordering.

### ¬ß2.4 ‚Äî Barrier-hit timestamping

`exit_ts = first_tick_ts where barrier triggered` (NOT mid-price ts). This is the **earliest possible** exit; engine attributes this trade to the test fold containing `t_start` (CPCV embargo guarantees `exit_ts < embargo_window` is held by upstream split logic, NOT enforced inside the closure).

**Anti-leak invariant:** the closure NEVER consults trade ticks beyond `t_vertical = 17:55 BRT same session as entry`. `exit_ts ‚àà [entry_ts, 17:55]`.

## ¬ß3 ‚Äî P60 / P20 / P80 percentile filter activation rules

**Source of truth:** spec yaml L80-82 + cpcv_harness.py:309-313 (anti-leak invariant docstring).

### ¬ß3.1 ‚Äî Filter calculation

`PercentileBands` come from `Percentiles126dState` (warmup builder, percentiles_126d_builder.py:91+). For the train fold producing the bands:

```
window = sorted({ DailyMetrics(d) for d in calendar.valid_sample_days
                                  if d in train_fold AND d < as_of_date })[-126:]
bands.magnitude     = (P20, P60, P80) over window
bands.atr_day_ratio = (P20, P60, P80) over window
```

**Critical anti-leak:** `as_of_date` for fold k is the **first session of fold k's test slice**, NOT the global in_sample start. Each fold sees a **distinct** `Percentiles126dState`. This is the **DEFERRED-T11 M1 fix** (¬ß6).

### ¬ß3.2 ‚Äî Activation triggers

Per spec yaml L80-81 + signal_rule.py:75-85:

| Trigger | Condition | Action |
|---------|-----------|--------|
| **Entry** | `Features.intraday_flow_magnitude > bands.magnitude.{p60|p50|p70}` (per trial) | Proceed to regime check |
| **Regime filter** (T1/T2/T3/T5 only ‚Äî T4 disables) | `bands.atr_day_ratio.p20 ‚â§ Features.atr_day_ratio ‚â§ bands.atr_day_ratio.p80` | Proceed to fade direction |
| **Direction** | per ¬ß1.3 | Emit Signal |
| **Failure (any)** | else | `Signal(FLAT, <reason>)` |

### ¬ß3.3 ‚Äî Tie-break determinism

`Percentiles126dBuilder._percentile` uses numpy type-7 linear interpolation (percentiles_126d_builder.py:139-153). Ties on bands are **deterministic** ‚Äî same input window ‚áí same bands ‚áí same signal. Closure preserves this; introduces no random shuffles, no `np.random` calls.

### ¬ß3.4 ‚Äî D-1 invariant

Per cpcv_harness.py:309-313 + percentiles_126d_builder.py:131-136 (`m.day < as_of_date` filter): bands are computed over days STRICTLY BEFORE the entry session. Day-of-entry is **not in the percentile window**, eliminating same-day leakage.

### ¬ß5.2 ‚Äî IR DARF treatment (Mira ruling)

Per atlas L210-225 (`treatment_choice: post_hoc_on_monthly_net_gain`):

**The closure does NOT compute IR DARF.** Per-trade PnL is **gross** of IR. Tax is applied at the Sharpe-distribution-aggregation layer (out of closure scope), preserving Lopez de Prado ¬ß3 DSR formula (gross-of-tax cohort statistics ‚Äî adding tax distorts the per-trade return distribution non-linearly).

**For T002.1.bis Beckett N6 verification (AC11):** Beckett's report N6 ¬ßX documents the gross PnL distribution AND a separate post-hoc table of monthly-aggregated IR-net PnL. Mira clearance Gate 4 metrics (DSR, PBO) are computed on **gross PnL** (research convention, Bailey-LdP 2014 ¬ß3); production capital ramp will use IR-net (Riven Gate 5 prep).

### ¬ß5.3 ‚Äî PnL formula (closure-internal)

Per trade `i`:

```
gross_pnl_rs[i] = (exit_price - entry_price) * sign(direction) * n_contracts * WDO_MULTIPLIER
                  - 2 * n_contracts * (brokerage_per_contract_side_rs + exchange_fees_per_contract_side_rs)
                  # slippage already baked into entry_price + exit_price via BacktestBroker._slippage()
```

**WDO_MULTIPLIER = R$ 10.00/ponto** ‚Äî single-source from `DOMAIN_GLOSSARY.md#wdo-mini-d√≥lar-futuro` per fix G005 (spec yaml L394). Hardcoded constant `WDO_MULTIPLIER = 10.0` already in exec_backtest.py:29 ‚Äî closure consumes verbatim.

### ¬ß5.4 ‚Äî Cost atlas SHA + version locks

Per `from_engine_config(...)` (exec_backtest.py:114-130): atlas SHA256 (LF-normalized) and `atlas_version` are validated at config-load time. Closure inherits the validated `BacktestCosts`. **NO atlas-side validation inside closure** ‚Äî done once at engine init.

## ¬ß6 ‚Äî Per-fold P126 rebuild architecture (DEFERRED-T11 M1 fix)

**Source of truth:** cpcv_harness.py:287-307 docstring + Aria 2026-04-26 architecture review finding M1.

This section is **architectural** ‚Äî Aria T0b sign-off **chooses among** the three options below. Mira's role here is to enumerate alternatives + state preference; Aria authority decides the canonical.

### ¬ß6.1 ‚Äî Mira preferred design: Option B (`backtest_fn_factory`)

**Signature:**

```python
def CPCVEngine.run(
    events: pd.DataFrame,
    splits: Sequence[CPCVSplit],
    backtest_fn_factory: Callable[[CPCVSplit, pd.DataFrame], Callable[[pd.DataFrame, pd.DataFrame, CPCVSplit], BacktestResult]],
    ...
) -> Sequence[BacktestResult]:
    for split in splits:
        train_events = events.iloc[split.train_indices]
        backtest_fn = backtest_fn_factory(split, train_events)   # ‚Üê per-fold closure
        results.append(backtest_fn(train_events, test_events, split))
```

**Where the factory is responsible for:**
1. Receiving `train_events` (rows of fold k's train slice).
2. Computing the per-day `DailyMetrics` over `train_events`.
3. Calling `Percentiles126dBuilder.build(metrics, as_of_date=first_test_session, ...)` to get the fold-specific `Percentiles126dState`.
4. Calling `make_backtest_fn(costs, calendar, percentiles_state)` to produce the closure.
5. Returning the closure.

**Why Mira prefers Option B:**
- **Engine becomes the orchestrator** of the per-fold lifecycle (single responsibility).
- **`make_backtest_fn` keeps its current pure-function signature** ‚Äî already passes `test_per_fold_p126_rebuild` because re-calling it produces a new closure with a new `Percentiles126dState`.
- **No change to `make_backtest_fn` signature** ‚áí minimal regression risk; existing tests stay green.
- **Factory pattern is canonical** in CPCV literature (Lopez de Prado AFML ¬ß7 and ¬ß12) ‚Äî N(splits) closures, each scoped to a single split.

### ¬ß6.2 ‚Äî Alternative design: Option A (`percentiles_provider` injected into closure)

**Signature:**

```python
def make_backtest_fn(
    costs: BacktestCosts,
    calendar: CalendarData,
    percentiles_provider: Callable[[pd.DataFrame], Percentiles126dState],
) -> Callable[..., BacktestResult]:
    def backtest_fn(train_events, test_events, split):
        # rebuild bands from THIS fold's train slice
        percentiles_state = percentiles_provider(train_events)
        ...
    return backtest_fn
```

**Tradeoffs vs Option B:**
- Pro: only one `make_backtest_fn` invocation per run; one closure; less factory-pattern overhead.
- **Con: rebuild happens INSIDE the closure body** ‚Äî closure is no longer pure (it computes percentiles) and is harder to unit-test (must mock the provider).
- Con: `make_backtest_fn` signature changes (breaks `test_per_fold_p126_rebuild` which currently re-instantiates).
- Con: `percentiles_state` enters and leaves the closure scope per call ‚Äî captures train_events implicitly per invocation, harder to reason about determinism.

### ¬ß6.3 ‚Äî Alternative design: Option C (state caller-managed, closure stateless)

**Signature:** `make_backtest_fn` keeps current signature **unchanged**; the **caller** (e.g., a new `run_5_trial_fanout_per_fold` orchestrator at cpcv_harness.py:475+) manually loops over splits, rebuilds `Percentiles126dState`, and calls `CPCVEngine.run` once per split with a freshly-instantiated closure.

**Tradeoffs:**
- Pro: zero changes to `CPCVEngine.run` signature; zero changes to `make_backtest_fn` signature.
- Con: `CPCVEngine.run` becomes a per-split helper, not a CPCV engine. Loses bulk parallelization potential.
- Con: caller has to manually orchestrate splits ‚Äî duplicated CPCV iteration logic across consumers.
- Con: weakest separation of concerns; engine no longer owns CPCV semantics.

### ¬ß6.4 ‚Äî Mira recommendation summary

| Option | Engine signature change | `make_backtest_fn` signature change | CPCV correctness | Mira preference |
|--------|------------------------|--------------------------------------|------------------|----------------|
| **B (factory)** | YES (`backtest_fn_factory` param) | NO | strongest | **PREFERRED** |
| A (provider injected) | NO | YES (breaking) | acceptable | secondary |
| C (caller-managed) | NO | NO | weakest | last resort |

**Aria T0b authority decides.** Mira spec is binding on ¬ß1-¬ß5, ¬ß7; binding on the existence of a per-fold rebuild mechanism; **non-binding** on which of A/B/C is canonical (Aria chooses).

## ¬ß7 ‚Äî Toy benchmark (Bailey-LdP 2014 reproducibility anchor)

**Source of truth:** Bailey & Lopez de Prado, "The Deflated Sharpe Ratio: Correcting for Selection Bias, Backtest Overfitting, and Non-Normality" (Journal of Portfolio Management, Vol. 40, No. 5, 2014), ¬ß3 (DSR formula derivation) + Table 2 (toy example calibration).

### ¬ß7.1 ‚Äî Toy strategy A: pure noise (no edge)

**Construction:**
- Synthetic per-trade returns drawn iid from `N(Œº=0, œÉ=1)`.
- N=45 paths √ó n_trades_per_path=20 (matches T002 CPCV 45-path √ó ~20 events/path order of magnitude).
- n_trials=5 (Bonferroni preserved).

**Bailey-LdP 2014 ¬ß3 expected metrics:**
- **`SR_estimate ‚âà 0.0`** ‚Äî noise has no skill; per-path Sharpe centered on zero.
- **`œÉ(SR_distribution) ‚âà 1/sqrt(n_trades) ‚âà 0.22`** ‚Äî sampling spread, not skill.
- **`DSR ‚âà 0.5`** ‚Äî by construction; null distribution gives 50% probability skill > 0.
- **`PBO ‚âà 0.5`** ‚Äî Bailey-Borwein-LdP 2014: under H0 (no skill), best-IS-Sharpe is uniformly distributed across rank-OOS positions ‚áí PBO = 0.5.

**Acceptance:** `make_backtest_fn` over toy strategy A reproduces these to within ¬±0.05 (sampling tolerance).

### ¬ß7.2 ‚Äî Toy strategy B: small edge

**Construction:**
- Synthetic per-trade returns drawn iid from `N(Œº=0.20, œÉ=1)` ‚Äî small positive edge (Sharpe ‚âà 0.20 per trade).
- N=45 paths √ó n_trades_per_path=20.
- Annualized SR (assuming 252 trading days, 1 trade/day): `SR_annual ‚âà 0.20 * sqrt(252) ‚âà 3.17`.

**Bailey-LdP 2014 ¬ß3 expected metrics:**
- **`SR_distribution_mean ‚âà 0.20`** per-trade (not annualized).
- **`DSR > 0.95`** with n_trials=5 Bonferroni (`p_adj = 0.01/5 = 0.002`) ‚Äî skill survives multiple-testing penalty.
- **`PBO < 0.25`** ‚Äî skill is robust across IS/OOS rank decomposition.

**Acceptance:** `make_backtest_fn` over toy strategy B reproduces these to within ¬±0.05.
